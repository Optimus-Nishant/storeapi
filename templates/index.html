API DOCUMENTATION

  • Introduction

    This is a store api written in python. As this service is only for authenticated user, so you have to register for the same. Once you register you will receive API ID and API Key which has to be provided for Authentication.(Api uses BasicAuth)
    API Requests, Url and Functions are descibed in the table below:-
    HTTP Request Url Function
    GET http://storeapi-snapego.rhcloud.com/store/api/items Will return a json containing all the items in the store
    POST http://storeapi-snapego.rhcloud.com/store/api/items Will add item to the store and give relevant resonse.Expects you to provide a json.
    PUT http://storeapi-snapego.rhcloud.com/store/api/items/(item id) Will update the Item associated with given item id.Expects you to provide a json.
    Delete http://storeapi-snapego.rhcloud.com/store/api/items/(item id) Will delete the Item associated with given item id.
    GET http://storeapi-snapego.rhcloud.com/store/api/items/search/(args) Will return a json containing search result

  • Responses

    Response code and thier Meanings are as follows.
    Response Code Meaning
    400 Bad Request
    200 Successfully Created
    204 Successfully Updated/Deleted.
    404 Not Found

  • Add Item

    Request Type: POST
    URL :http://storeapi-snapego.rhcloud.com/store/api/items
    Content Type:application/json
    Response on Sucess: Created Item Json and Response 201.

    Example of Json: 

    	
	{
	  "name": "Item One",
	  "description": "This is First Item",
	  "price": 3000,
	  "quantity": 11,
	  "seller": "Nishant",
	  "category": "ABC"
	}

**name, description and price are compulsory elements

  • Updating Element

    Request Type: PUT
    URL :http://storeapi-snapego.rhcloud.com/store/api/items/(item_id)
    Content Type:application/json
    Response on Sucess: Updated Item Json and Response Code 204.

    Example of Json: 

    	
	{
	  "description": "This item is updated with new price and category",
	  "price": 4000,
	  "quantity": 9,
	  "seller": "New Seller",
	  "category": "New Category"
	}

**You are supposed to send the keys that you want to update. 
Nothing is compulsary but empty json will raise bad request.
In this exampe I am not updating item name.

  • Searching Element

    Request Type:GET
    URL :http://storeapi-snapego.rhcloud.com/store/api/items/search/(args)
    Response on Sucess: Search Result.

    Search is supported for item name, category, seller , priceorder and their combinations.
    For low to high price : lowtohigh=1
    For high to low price : hightolow=1
